# sql-football-league

6-a-Side Company Football League Database
Submission Guide & Navigation README

Author: Taline Manuelian

Software Requirements:
-PostgreSQL
-pgAdmin

BRIEF DESCRIPTION:

This submission contains the complete implementation of a relational database system for managing a 6-a-side Company Football League. The benefit of running this sample is to demonstrate both correct relational design principles and practical SQL implementation aligned with the project brief.

The following files are included in this submission:

- SQL SCRIPT: Taline_Manuelian_SQL_DatabaseImplementation_FootballLeague_Script.sql
- DOCUMENTATION: Taline_Manuelian_SQL_DatabaseImplementation_FootballLeague_Documentation.pdf
- VIDEO PRESENTATION: Taline_Manuelian_SQL_DatabaseImplementation_FootballLeague_VideoPresentation.mp4
- SQL ERD: Taline_Manuelian_SQL_DatabaseImplementation_FootballLeague_ERD
- EXCEL TABLES: Taline_Manuelian_SQL_DatabaseImplementation_FootballLeague_ExcelDataReorganisation.xlsx

ORDER OF ACCESS - HOW TO NAVIGATE THIS SUBMISSION:

1) README File (Navigation Guide)

This README file explains:

-File structure
-Execution order
-Database setup instructions
-Key implementation highlights

This should be used as a reference guide while reviewing the project.

2) Video Presentation (Overview)

Begin with the recorded presentation, which includes:

-Overview of the task and objectives
-Database design explanation
-ER diagram walkthrough
-DDL and DML implementation summary
-Testing explanation
-Live demonstration in pgAdmin
-Reflection, conclusions, and future improvements

This provides both conceptual explanation and practical validation of the working database.

3) Documentation (Technical Explanation)

After watching the presentation, review the full documentation for deeper technical detail. This includes:

-ER Diagram (Crow’s Foot notation)
-Data Dictionary
-Assumptions and Constraints (per table)
-Functional dependencies
-Normalisation justification (3NF)
-DDL explanation
-DML explanation
-Testing section with expected and actual outcomes

This document explains the reasoning behind all implementation decisions.

4) SQL Script (Full Implementation)

After reviewing the documentation, open and run the SQL script.

The script contains:

-DDL (CREATE TABLE statements with constraints)
-DML (INSERT statements for reference and transactional data)
-Test queries used in the presentation
-Additional tested queries

Running this file recreates the entire database from scratch.

Running the Script:

Step 1 - Create a new database
Step 2 - Open the .sql file: Taline_Manuelian_SQL_DatabaseImplementation_FootballLeague_Script.sql
Step 3 - Execute the script in full.
Step 4 - Run the test case queries included at the bottom of the script.
Step 5 - Compare results with documentation screenshots.

5) Excel Tables (Data Reorganisation)

Finally, review the Excel file, which contains:

-Reorganised data structured according to the final relational schema
-Tables separated according to normalised design in 3NF

This demonstrates how raw scenario data was transformed into a structured relational model.
